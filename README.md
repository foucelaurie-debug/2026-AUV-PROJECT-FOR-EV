"# AUV \"Cruise\" 项目文档" 
"## 1. 项目概述" 
\"Cruise\"项目是一个为水下机器人（AUV）设计的软件系统，基于ROS2框架开发。该系统集成了硬件控制、图像采集、AI目标检测等多个核心功能模块，旨在实现AUV的自主导航与环境感知。 
"## 2. 核心功能模块" 
"### 2.1 硬件管理单元 (`uv_hm`)" 
作为系统的“小脑”，硬件管理单元负责与AUV的底层硬件进行直接交互。 
"- **节点**: `uv_hmu.py`" 
"- **功能**:" 
"  - 通过串口与H750、F407等微控制器通信，发送控制指令并接收传感器数据。" 
  - 实现PID闭环控制，根据上层指令和传感器反馈，精确控制AUV的运动。 
"  - 管理和配置PID参数、推进器推力曲线等，以适应不同任务需求。" 
"  - 控制舵机、LED灯、电磁铁等外围设备。" 
"### 2.2 视觉处理模块 (`uv_vision`)" 
该模块是AUV的“眼睛”，负责从摄像头获取原始图像数据。 
"- **节点**: `uv_capture.py`" 
"- **功能**:" 
"  - 支持前视和下视两组立体摄像头。" 
"  - 捕获图像，并根据提供的参数进行立体校正。" 
"  - 将原始和校正后的图像（左、右、全景）发布为ROS话题，供AI模块使用。" 
"  - 定期将图像帧保存到本地文件系统，用于调试和数据采集。" 
"### 2.3 AI视觉检测模块 (`uv_ai`)" 
这是AUV的“大脑”，负责对视觉信息进行分析和理解。 
"- **节点**: `uv_detect_demo.py`" 
"- **功能**:" 

"  - 订阅视觉处理模块发布的图像话题。" 
"  - 使用YOLOv8模型进行实时目标检测。" 
"  - 将检测结果（包含目标类别、位置、置信度等信息）发布为自定义的ROS消息和ROS服务。" 
"  - 将带有检测框的图像发布出去，便于调试和可视化。" 
"## 3. 系统架构" 
系统采用分层架构，通过ROS2的话题和服务进行模块间的解耦和通信。 
"1. **感知层 (`uv_vision`)**: 负责从摄像头获取原始数据。" 
"2. **决策层 (`uv_ai`)**: 分析图像数据，进行目标识别，并根据任务需求生成决策指令。" 
"3. **执行层 (`uv_hm`)**: 接收决策指令，并将其转换为对底层硬件的精确控制。" 
"## 4. 其他说明" 
"- 项目中包含了大量自定义的ROS消息类型 (`uv_msgs`)，用于在不同模块间传递复杂的数据结构。" 
"- 代码中包含了佛祖保佑和猫猫图案等有趣的注释，为项目增添了一丝独特的色彩。"
